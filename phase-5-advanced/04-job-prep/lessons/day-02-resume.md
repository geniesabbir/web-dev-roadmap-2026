# Day 2: Crafting a Resume That Gets Interviews

## Introduction

Your resume has about 6-10 seconds to make an impression. In that brief window, a recruiter decides whether to keep reading or move on. Today, we'll craft a resume that passes ATS (Applicant Tracking Systems), catches recruiters' attention, and effectively communicates your value as a developer.

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Resume Journey                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│    Your Resume                                                          │
│        │                                                                │
│        ▼                                                                │
│   ┌─────────────┐     70% of resumes                                   │
│   │    ATS      │────────────────────▶  Rejected                       │
│   │   Filter    │     don't make it                                    │
│   └──────┬──────┘                                                      │
│          │                                                              │
│          ▼ Passes ATS                                                   │
│   ┌─────────────┐     6-10 second                                      │
│   │  Recruiter  │────────────────────▶  "Maybe" Pile                   │
│   │    Scan     │     initial scan                                     │
│   └──────┬──────┘                                                      │
│          │                                                              │
│          ▼ Catches Interest                                             │
│   ┌─────────────┐                                                      │
```

By the end of this lesson, you will:

## 1. Understanding ATS Systems

### How ATS Works

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    ATS Processing Flow                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. PARSING                                                             │
│     ┌─────────────────────────────────────────────────────────────┐    │
│     │  ATS extracts text from your resume                          │    │
│     │  • Contact information                                       │    │
│     │  • Work history                                              │    │
│     │  • Education                                                 │    │
│     │  • Skills                                                    │    │
│     │                                                              │    │
│     │  ⚠️  Fancy formatting can break parsing!                      │    │
│     └─────────────────────────────────────────────────────────────┘    │
│                                                                         │
│  2. KEYWORD MATCHING                                                    │
│     ┌─────────────────────────────────────────────────────────────┐    │
│     │  System compares your resume to job description              │    │
│     │  • Required skills: React, Node.js, TypeScript               │    │
│     │  • Experience level: "3+ years"                              │    │
│     │  • Job titles: "Software Engineer"                           │    │
│     │                                                              │    │
│     │  ⚠️  Missing keywords = lower ranking                         │    │
│     └─────────────────────────────────────────────────────────────┘    │
│                                                                         │
│  3. RANKING                                                             │
│     ┌─────────────────────────────────────────────────────────────┐    │
│     │  Resumes scored and ranked by match percentage               │    │
│     │  Top candidates surfaced to recruiters                       │    │
│     │                                                              │    │
│     │  Typical threshold: 70-80% match to be considered            │    │
│     └─────────────────────────────────────────────────────────────┘    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### ATS-Friendly Formatting

```
DO:
✓ Use standard section headings (Experience, Education, Skills)
✓ Use simple fonts (Arial, Calibri, Georgia)
✓ Save as PDF (maintains formatting) or .docx
✓ Use standard bullet points (•, -, *)
✓ Put dates in consistent format (Jan 2023 - Present)
✓ Include full job titles and company names

DON'T:
✗ Use tables or columns (parsing issues)
✗ Put important info in headers/footers
✗ Use images, icons, or graphics
✗ Use text boxes
✗ Use creative fonts
✗ Abbreviate everything (use "JavaScript" not "JS" at least once)
```

## 3. Writing Powerful Bullet Points

### The XYZ Formula

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    XYZ Formula for Bullet Points                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   "Accomplished [X] as measured by [Y], by doing [Z]"                   │
│                                                                         │
│   COMPONENTS:                                                           │
│   X = What you accomplished (the result)                                │
│   Y = How it was measured (the metric)                                  │
│   Z = How you did it (the method/technology)                            │
│                                                                         │
│   EXAMPLE:                                                              │
│   ┌─────────────────────────────────────────────────────────────────┐  │
│   │  "Reduced page load time by 60% by implementing lazy loading,   │  │
│   │   image optimization, and code splitting in a Next.js app"      │  │
│   │                                                                 │  │
│   │   X = Reduced page load time                                    │  │
│   │   Y = by 60%                                                    │  │
│   │   Z = implementing lazy loading, image optimization,            │  │
│   │       code splitting in Next.js                                 │  │
│   └─────────────────────────────────────────────────────────────────┘  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Before and After Examples

```
WEAK BULLET POINTS:

❌ "Worked on the frontend using React"
   → What did you actually accomplish?

❌ "Responsible for maintaining the codebase"
   → That's just a job description

❌ "Helped improve website performance"
   → By how much? How did you do it?

❌ "Used Git for version control"
   → This is expected, not an achievement


STRONG BULLET POINTS:

✅ "Built a customer dashboard using React and TypeScript that reduced
    support tickets by 35% by enabling self-service account management"

✅ "Architected and implemented a caching layer with Redis that decreased
    API response times from 800ms to 50ms, improving user experience for
    10,000+ daily active users"

✅ "Led migration from JavaScript to TypeScript across 50,000+ lines of
    code, reducing production bugs by 40% and improving developer onboarding
    time by 2 weeks"

✅ "Designed and deployed a CI/CD pipeline using GitHub Actions that reduced
    deployment time from 45 minutes to 8 minutes and enabled 5x more frequent
    releases"
```

```
│  • Bundle size reductions                                               │
│  • Database query optimizations                                         │
│                                                                         │
│  SCALE METRICS                                                          │
│  • Number of users served                                               │
│  • Requests per second handled                                          │
│  • Data processed (GB, records, transactions)                           │
│  • Uptime percentage                                                    │
│                                                                         │
│  PRODUCTIVITY METRICS                                                   │
│  • Development time saved                                               │
│  • Deployment frequency increases                                       │
│  • Bug reduction percentages                                            │
│  • Code coverage improvements                                           │
│                                                                         │
│  BUSINESS METRICS                                                       │
│  • User engagement increases                                            │
│  • Conversion rate improvements                                         │
│  • Support ticket reductions                                            │
│  • Revenue impact (if appropriate to share)                             │
│                                                                         │
│  PROJECT SCOPE                                                          │
│  • Lines of code written/refactored                                     │
│  • Number of features shipped                                           │
```

### Organizing Your Skills

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Skills Section Format                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  GOOD FORMAT (Category-based):                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │

│  │  Frontend: React, Next.js, Redux, Tailwind CSS, React Query      │   │
│  │  Backend: Node.js, Express, PostgreSQL, MongoDB, Redis           │   │

│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  ALTERNATIVE (Proficiency-based - use sparingly):                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Proficient: React, TypeScript, Node.js, PostgreSQL              │   │
│  │  Experienced: Python, AWS, Docker, GraphQL                       │   │
│  │  Familiar: Go, Kubernetes, Terraform                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  AVOID:                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  ✗ Skill bars or percentages (meaningless)                       │   │
│  │  ✗ Rating yourself "JavaScript: 9/10"                            │   │
│  │  ✗ Listing soft skills (Communication, Teamwork)                 │   │
│  │  ✗ Including obvious things (Microsoft Office, Google)           │   │
│  │  ✗ Outdated technologies (jQuery, AngularJS, unless relevant)    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

```
│  ✗ Irrelevant experience taking up space                               │

│  ✗ References available upon request (assumed)                         │
│                                                                         │
│  STRATEGIC MISTAKES                                                     │
│  ✗ Same resume for every application                                   │
│  ✗ Not matching keywords from job description                          │
```

## 7. Resume Tailoring Strategy

### For Each Application

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Resume Tailoring Checklist                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. ANALYZE THE JOB POSTING                                             │
│     □ Highlight required skills and technologies                        │
│     □ Note preferred/nice-to-have skills                                │
│     □ Identify key responsibilities                                     │
│     □ Understand the company/team context                               │
│                                                                         │
│  2. REORDER YOUR SKILLS                                                 │
│     □ Put matching required skills first                                │
│     □ Include exact technology names from posting                       │
│     □ Remove irrelevant skills to make space                            │
│                                                                         │
│  3. ADJUST YOUR BULLET POINTS                                           │
│     □ Lead with most relevant experience for this role                  │
│     □ Emphasize matching technologies and achievements                  │
│     □ Use similar language to job posting                               │
│                                                                         │
│  4. UPDATE SUMMARY (if you have one)                                    │
│     □ Align with role requirements                                      │
│     □ Highlight relevant expertise                                      │
│                                                                         │
│  5. FINAL CHECK                                                         │
│     □ Run through ATS checker (jobscan.co)                              │
│     □ Proofread for errors                                              │
│     □ Ensure links work                                                 │
│     □ Save with clear filename: FirstName_LastName_Resume.pdf           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
