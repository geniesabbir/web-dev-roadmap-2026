# Phase 4: DevOps & Deployment

**Duration:** 4-6 weeks
**Goal:** Deploy and maintain production applications

## Overview

This phase teaches you to take applications from your local machine to production. You'll learn containerization, CI/CD, cloud deployment, and monitoring.

## Prerequisites

Before starting this phase, ensure you:
- Completed Phases 1-3
- Have a full-stack application ready to deploy
- Basic command line proficiency

## Sections

### 1. [Linux & CLI](./01-linux-cli/README.md) (1 week)
- Essential Linux commands
- File permissions
- Process management
- SSH and remote access
- Shell scripting basics

### 2. [Docker](./02-docker/README.md) (1-2 weeks)
- Container concepts
- Dockerfile creation
- Docker Compose
- Multi-stage builds
- Container networking
- Volume management

### 3. [CI/CD](./03-cicd/README.md) (1 week)
- GitHub Actions
- Automated testing
- Automated deployments
- Environment management
- Secrets handling

### 4. [Cloud Deployment](./04-cloud/README.md) (1-2 weeks)
- Platform options overview
- Vercel (Frontend/Next.js)
- Railway/Render (Backend)
- AWS basics (EC2, S3, RDS)
- Domain and SSL setup

## Phase 4 Projects

Deploy your Phase 2 and 3 projects:
1. Deploy Next.js app to Vercel
2. Deploy backend API to Railway
3. Set up CI/CD pipeline
4. Configure custom domain
5. Set up monitoring

- [ ] Complete Docker

- [ ] Complete Cloud Deployment
